# 一笔 — 记账软件需求报告

## 1 项目概述

本项目旨在开发一款名为"一笔"的面向安卓平台的轻量级记账应用（"收支"指现金及电子账户的流入与流出，不含虚拟货币交易），功能聚焦于核心记账操作：记录收支、分类管理及数据统计概览。目标产出为可打包为`.apk`安装包的原生安卓应用，最低兼容API 26+，采用Kotlin语言与Jetpack Compose框架开发。本项目不包含进阶功能，但需预留扩展接口以支撑后续功能迭代。

## 2 目标用户

主要面向具备基础智能手机操作能力的个人用户，尤其是有日常收支记录需求但无需复杂财务管理的学生及职场新人。用户期望通过简洁的操作流程快速完成记账，并以直观方式查看消费趋势。

## 3 功能需求列表

### 3.1 收支记录模块

**是否可扩展：是**。接口约定：`interface IncomeExpenseRecord : ExpandableUnit { fun recordTransaction(type: TransactionType, amount: Double, categoryId: String, date: LocalDate): String }`。该模块负责录入单笔收入或支出交易，包含金额、类别、日期、备注字段，支持新增、编辑、删除操作。

### 3.2 分类管理模块

**是否可扩展：是**。接口约定：`interface CategoryManagement : ExpandableUnit { fun addCategory(name: String, type: TransactionType, icon: String): String }`。该模块维护收入与支出的分类目录，支持增删改查操作，预置餐饮、交通、工资等基础分类，允许用户自定义分类名称及图标。

### 3.3 数据统计概览模块

**是否可扩展：否**。该模块按日、周、月维度聚合收支数据，以饼图展示分类占比，以折线图展示收支趋势，提供支出排行列表。数据源自本地数据库查询，不涉及外部服务调用。

## 5 技术选型建议

### 5.1 架构与框架

- **开发语言**：Kotlin
- **UI框架**：Jetpack Compose（声明式UI，"声明式UI"指通过描述界面状态而非命令式操作来构建UI的范式）
- **架构模式**：MVVM + Repository模式，模块间通过接口解耦
- **本地数据库**：Room（SQLite封装）
- **依赖注入**：手动注入（无需第三方框架，ViewModel构造时直接传入Repository实例）
- **异步处理**：Kotlin Coroutines + Flow
- **部署方式**：Gradle构建APK后直接安装至Android设备，无需服务端，全部数据存储于本地Room数据库

### 5.2 可扩展性设计示例

**实现语言：Kotlin**

```kotlin
/**
 * 可扩展单元基接口，所有可扩展模块均实现此接口。
 * 后续扩展功能（预算提醒、多币种、账单导入）通过实现此接口接入。
 */
interface ExpandableUnit {
    /** 模块唯一标识 */
    val moduleId: String
    /** 模块显示名称 */
    val displayName: String
    /** 初始化模块资源 */
    fun onInitialize(context: ModuleContext)
    /** 销毁模块资源 */
    fun onDestroy()
}

/**
 * 收支记录扩展接口，定义交易录入的扩展点。
 */
interface IncomeExpenseRecord : ExpandableUnit {
    override val moduleId: String get() = "income_expense_record"
    override val displayName: String get() = "收支记录"

    override fun onInitialize(context: ModuleContext) {
        // 初始化数据库连接及分类缓存
    }

    override fun onDestroy() {
        // 释放数据库连接
    }

    /**
     * 记录一笔交易，返回交易ID。
     * @param type 收入或支出
     * @param amount 交易金额，单位：元
     * @param categoryId 分类ID
     * @param date 交易日期
     * @return 交易唯一标识
     */
    fun recordTransaction(
        type: TransactionType,
        amount: Double,
        categoryId: String,
        date: LocalDate
    ): String

    /**
     * 删除指定交易。
     */
    fun deleteTransaction(transactionId: String)
}

/** 交易类型枚举 */
enum class TransactionType { INCOME, EXPENSE }

/** 模块上下文，提供模块运行所需的公共资源 */
data class ModuleContext(
    val databaseProvider: DatabaseProvider,
    val resourceProvider: ResourceProvider
)
```

#### 调用示例

**场景**：用户在主界面点击"记一笔"按钮后，系统调用`IncomeExpenseRecord.recordTransaction()`完成交易录入。调用流程为：Compose UI层收集用户输入，ViewModel调用模块接口，模块内部持久化至Room数据库，返回交易ID，UI刷新列表。

```kotlin
// 以下为可编译的调用示例代码
class RecordViewModel(
    private val recordModule: IncomeExpenseRecord
) : ViewModel() {

    fun recordExpense(
        amount: Double,
        categoryId: String,
        date: LocalDate
    ) {
        viewModelScope.launch {
            val transactionId = recordModule.recordTransaction(
                type = TransactionType.EXPENSE,
                amount = amount,
                categoryId = categoryId,
                date = date
            )
            // transactionId 用于UI刷新及后续编辑
        }
    }
}
```
